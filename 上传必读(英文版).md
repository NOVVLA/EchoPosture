# Upload Read-Me（Remote Upload Rules）

This document defines the minimum rules for uploading files from the local EchoPosture workspace to a remote Git repository. Read it together with `ROE.md` (Rules of Engagement) and `PROCESS_AUDIT.md` (Process Audit Rules).

## Default Upload Policy

Upload only source files, intentional documentation, tests, and other artifacts that are part of the product or are explicitly requested for publication. A file being present on disk does not make it eligible for upload.

The following are normally local-only and must not be uploaded:

- Local operation or maintainer files, such as `README.local.md`, machine-specific notes, temporary checklists, and workspace instructions.
- Untracked files whose purpose, ownership, or intended destination has not been confirmed.
- Generated or environment-specific content: `runtime/`, `dist/`, `logs/`, `_backups/`, `.downloads/`, `.uv-cache/`, `__pycache__/`, build outputs, `.exe`, `.obj`, `.pdb`, temporary previews, and review folders.
- Private documents, screenshots, recordings, personal paths, credentials, API keys, tokens, cookies, and diagnostic logs containing user data.

Ignored files are not automatically safe to upload, and untracked files are not automatically required. If an untracked file is intentionally part of the product, document why it is needed and obtain explicit approval before adding it.

## Required Pre-Upload Review

Before staging anything:

1. Run `git status --short` and identify every changed, deleted, and untracked path.
2. Inspect `git diff` and confirm that no unrelated, frozen, generated, or local-only file is included.
3. Stage named paths explicitly; do not use `git add -A` or `git add .` when the workspace contains local work.
4. Run the smallest relevant tests, lint checks, build checks, or documentation checks.
5. Review staged content with `git diff --cached`, then confirm the target branch and canonical `origin` remote.

## Commit and Push

Use a clear commit subject describing the real change. Record required verification, risks, skipped checks, and artifact hashes in `DEVELOPMENT_LOG.md`. Push only the reviewed commit to the intended remote branch. If the upload fails, report the failure and local commit SHA; never claim that a local commit was successfully delivered.

When uncertain whether a file belongs in the remote repository, leave it unstaged and ask for confirmation.
