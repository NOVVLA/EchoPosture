# EchoPosture (潜影护脊)

Windows 桌面端姿态调试原型。

## 现在能做什么

- 实时显示摄像头画面
- 在画面上叠加眼睛、肩膀、头颈轴线
- 显示模型输出的可读状态
- 一键校准当前舒适坐姿
- 切换高性能模式（72 FPS）
- 独立测试遮罩层和纯视觉输出

## 启动方式

推荐从这个 ASCII 入口打开：

- `C:\Users\aaabb\Documents\ICC`

双击：

- [run_debug_ui.cmd](run_debug_ui.cmd)

这是当前最完整的入口。

## 调试 UI

窗口右侧会显示：

- `脸部距离`
- `肩膀倾斜`
- `当前基准`
- `状态`
- `原因`

左侧是摄像头实时画面，画面上会叠加：

- 双眼中心点和连线
- 左右肩点和肩线
- 鼻子点
- 鼻子到双肩中心的头颈轴线

## 校准流程

1. 让笔记本屏幕处在你觉得舒服的位置。
2. 坐直，保持自然使用电脑时的姿势。
3. 点击右侧 `校准当前姿势`。
4. 之后程序会把这组视觉数据当成健康基准。

## 高性能模式

右侧有一个复选框：

- `高性能模式（72帧捕捉用于高流畅度）`

勾选后会把 UI 刷新和摄像头采集尽量切到 72 FPS。  
这更流畅，但 CPU 占用会更高。

## 运行原理

项目内已经打包了一份 Python 3.11 运行环境，放在 `runtime/python311`。  
`Documents\ICC` 是一个纯英文路径入口，指向当前项目文件。  
启动脚本会优先从这个入口运行，避开中文路径下 MediaPipe 的资源加载问题。

## 常见问题

### 双击后一闪而过

说明启动失败。优先检查：

- 摄像头是否被别的程序占用
- 是否从 `C:\Users\aaabb\Documents\ICC` 入口启动
- 是否误删了 `runtime/python311`

### 画面里没看到肩膀

摄像头视野太窄、光线太暗、或者你离镜头太近时都可能这样。  
这时程序仍可能只根据脸部距离工作。

### 为什么有些时候状态是 `GOOD_PART`

表示当前只拿到了部分可用指标。  
例如脸部可见，但肩膀暂时没进入画面。
